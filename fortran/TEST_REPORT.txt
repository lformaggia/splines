================================================================================
IDENTIFIED BUGS
================================================================================

BUG #1: Array Indexing Error in trcoe.f (Line 262)
Severity: CRITICAL
File: trcoe.f, Line 262
Description:
    In the quadratic end condition handler, the code sets c(1)=0.d0 when it
    should set c(n)=0.d0. This corrupts the first element of the upper
    diagonal array when trying to set the last element.

    Current (WRONG):
        Line 260-262:
             a(n)=1.d0
             b(n)=1.d0
             c(1)=0.d0      <-- Should be c(n)

    Corrected code should be:
        Line 260-262:
             a(n)=1.d0
             b(n)=1.d0
             c(n)=0.d0      <-- Fixed

Impact: Any spline with quadratic boundary conditions will fail
Fix: Change c(1) to c(n)

================================================================================
BUG #2: Arc Length Computation Error
Severity: CRITICAL
Files: slen.f, coeff.f, lengt.f
Description:
    Arc lengths computed are systematically 30-100% larger than expected,
    even for simple straight lines where arc length should equal straight-line
    distance.

    The issue appears to be in how the polynomial coefficients for ||x,u||^2
    are computed in coeff.f, or possibly in how the integration is performed
    in lengt.f.

    For straight lines:
    - Expected: 5.0
    - Computed: 6.555
    - Error: +31.1%

    For curves:
    - Integration fails to converge ("QROMBS: too many steps")
    - Results are NaN

Root Cause: Unknown - requires detailed investigation of:
    1. Polynomial coefficient calculation in coeff.f
    2. Integration routine in lengt.f
    3. Interaction between tangent vectors and arc length computation

Impact: All arc length calculations are unreliable
Status: Needs investigation of mathematical formulas in coeff.f

================================================================================
BUG #3: NaN/Infinity in Tangent Vectors for Circular Curves
Severity: CRITICAL
File: Likely in evtan.f or trcoe.f
Description:
    When computing tangent vectors for circular spline curves, some tangent
    values become NaN or Infinity, which then propagates to arc length and
    curvature calculations.

    Example from test output:
    t(:, 1) = ( -0.18970221, ************), mag=    Infinity
    t(:, 2) = ( -0.20586655, ************), mag=    Infinity

    This does NOT occur for:
    - 2D straight lines
    - Parabolic curves
    - 3D straight lines (though arc lengths are wrong)

Root Cause: Likely uninitialized variables or numerical singularity in the
           tridiagonal solver (trif/tris2) when dealing with curved geometry

Impact: Cannot compute spline tangents for circular/curved geometry
Status: Requires debugging of evtan/trcoe/trif/tris2

================================================================================
BUG #4: Surface Chord Length Computation
Severity: HIGH
File: getta3.f
Description:
    The getta3 subroutine for surface interpolation reports incomplete
    chord length computation.

    Test output: "Chord lengths not properly computed"

    This suggests that the choret array (chord lengths on iso-parametric
    curves) is not being properly populated, even though the routine
    appears to set these values at lines 172 and 215.

Root Cause: Possible array indexing issue or incomplete initialization
Impact: Surface splines cannot be properly constructed
Status: Needs investigation of array storage and computation in getta3
