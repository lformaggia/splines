BUG DOCUMENTATION (Fix Recommendations)
================================================================================

1. fix_trcoe_quadratic_bug.txt (4.4 KB)
   ★ CRITICAL - OBVIOUS BUG ★

   Location: trcoe.f, line 262
   Bug: c(1)=0.d0 should be c(n)=0.d0
   Severity: CRITICAL

   This is a simple array indexing error in the quadratic boundary condition
   setup that corrupts the tridiagonal system. ONE-CHARACTER FIX.

   Impact: Any spline with quadratic boundary conditions will fail
   Affected: ispty(1)=4 or ispty(2)=4

   Proposed Fix: Change line 262 from:
       c(1)=0.d0
   To:
       c(n)=0.d0

2. fix_arc_length_investigation.txt (5.0 KB)
   Detailed investigation of arc length errors (30-100% too large)

   Issue: Arc lengths computed are systematically larger than expected
   Example: 3D straight line expected 5.0, computed 6.555 (+31%)

   Possible root causes:
   - Tangent vector scaling incorrect
   - Polynomial coefficients in coeff.f wrong
   - Romberg integration has issues

   Includes debugging strategy and mathematical formulas for verification

3. fix_nan_infinity_investigation.txt (6.4 KB)
   Detailed investigation of NaN/Infinity in tangent vectors

   Issue: Tangent vectors become NaN or Infinity for curved geometry
   Observation: Problem is dimension and geometry dependent

   Possible root causes:
   - Uninitialized variables in tridiagonal solver
   - Numerical instability in Romberg integration
   - Interaction between tangent and arc length computation

   Includes debugging strategy and safety measures
